# 8Caps Marketing Directory Website - Project Context

## Project Goal

Build a marketing website for **8Caps** that acts as a credibility hub and service directory for all websites, brands, tools, and services owned by 8Caps.

The purpose of the website is to:

1. Establish credibility for the 8caps domain when used for outreach.
2. Let visitors explore all services owned or operated by 8Caps.
3. Give each website/service its own dedicated profile page.
4. Allow admin users to add, edit, and manage sites from a private dashboard.
5. Include a future-ready onboarding system where an admin can submit a website URL, and the platform crawls/analyzes the site to generate a draft overview, service list, tags, and suggested content.

---

# Core Concept

The website is essentially a directory of 8Caps-owned websites.

Example structure:

- Homepage
- Services / Websites Directory
- Individual Website Profile Pages
- About 8Caps
- Contact / Enquiry Page
- Admin Dashboard
- Admin Login
- Add New Website Flow
- Edit Website Content Flow

Each website listed should have its own page containing:

- Website name
- URL
- Logo
- Short summary
- Full overview
- Main services
- Target audience
- Industry/category
- Key benefits
- Screenshots/images
- CTA button to visit the site
- CTA button to enquire through 8Caps
- SEO metadata

---

# Tech Direction

Use a modern full-stack web app stack.

Preferred direction:

- Next.js / React
- TypeScript
- Tailwind CSS
- Supabase for database and authentication
- Supabase Storage or local media handling for logos/screenshots
- Server-side routes/actions for crawling and AI analysis
- Optional: OpenAI/LLM integration for content generation

The site should be clean, modern, fast, credible, and simple to maintain.

# Website Onboarding Flow

The key feature is the ability to onboard a new site from a URL.

## Flow

Admin enters:

- Website URL
- Optional notes
- Optional category
- Optional ownership/status

The system should then:

1. Crawl the homepage
2. Extract page title, meta description, headings, visible text, and links
3. Optionally crawl a limited number of internal pages
4. Analyze the content using an LLM
5. Generate a draft website profile
6. Save it as a draft for admin review
7. Allow admin to edit and publish

---

# Generated Fields

The analysis system should generate:

- Website name
- Short summary
- Full overview
- Services offered
- Target audience
- Industry/category suggestions
- Keywords/tags
- Suggested CTA text
- Suggested SEO title
- Suggested SEO description
- Suggested slug
- Confidence score
- Notes about missing/unclear information

Important:

Generated content should NEVER auto-publish.

All generated content should be saved as a draft requiring admin review.

---

# Crawling Rules

Keep crawling simple and safe initially.

## Initial Version

- Crawl only the submitted URL
- Extract readable text
- Ignore scripts/styles
- Store raw extracted text for debugging
- Limit content length
- Handle errors gracefully

## Future Version

- Crawl up to 5 internal pages
- Prioritize:
  - About
  - Services
  - Pricing
  - Contact
  - Features
- Respect rate limits
- Avoid aggressive crawling

# Claude Code Instructions

Start by creating the project structure and building Phase 1 using clean placeholder data.

Use reusable components such as:

- SiteCard
- SiteHero
- CategoryFilter
- DirectoryGrid
- CTASection
- AdminLayout
- SiteForm
- StatusBadge

Keep the code clean and scalable.

Do NOT hardcode everything into pages.

Use typed data models from the start so the project can later move from mock data to Supabase easily.

Use TypeScript types/interfaces for:

- Site
- Service
- Category
- Screenshot
- Tag

Do not implement the AI crawler in the first commit unless the base site and admin structure are complete.
